Introduction

The game engine consists of numerous files. The details of how the files work is beyond the scope of these tutorials. You can download a bundle of the files here.

Your game engine will be initiated by an HTML file referencing JavaScript in three sections.

  • Body tag - This will have a JavaScript snippet executed onload. This will create a new instance of the Scene object (actually, a subclass of Scene that you will create) that will be sent into the game engine.
  • JavaScript include files - These will be all of the files that are part of the engine and any files you create. Note: The order that these are listed in is important. Modifying this order could break things.
  • Canvas DOM object - This is the section of the web page where all WebGL activity will be encapsulated. It identifies the pixel width and height where the game will be rendered.

onload

The entry point for your code will be an object you create that is a subclass of the engine's "Scene" object. The object type you are instantiating a subclass of Scene that you will create (see tutorial 1).

<body onload="
	var mainGame = new MyGameScene();
	gEngine.Core.initializeEngineCore('GLCanvas', mainGame);
">

JavaScript Includes

This code can be dropped into your HTML file as-is for all cases. The last line is an exception. This is where your custom code needs to be added. If you have multiple files, you need an include for each file. Again, the order is important so be sure you know what order things are being referenced.
