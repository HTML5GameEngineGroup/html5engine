Chapter 6: Defining Behaviors and Detecting Collisions

  • 6.1. Game Object Abstraction
    • WASD: to move Dye. Yeah, boring.
  • 6.2. Vectors: Front Direction and Chase
    • H: drive the brain with arrows
    • J: WASD move Dye and Brain points to Dye immediately
      • Oscillates when Brain eventually gets to Dye
    • K: moves Dye and Brain points to Dye gradually/smoothly
      • Orbits around a small circle when catches up to Dye
    • Would be nice if we can detect collision and stop
  • 6.3. Bounding Box and Collisions
    • Exactly the same control as above
    • Brain: now stops when it touches Dye's bound (not touching Dye)
    • Dye touches WC bound is detected (nothing is done yet)
  • 6.4. Per Pixel Collisions
    • WASD: move one
    • Arrows: move another
    • When pixel touches, shows DyePack at the first intersection found.
  • 6.5. Per Pixel Collisions with Rotated Image
    • Same behavior as above
    • E/P: keys rotate the texture, behavior exactly the same
  • 6.6. Sprite Per Pixel Collisions
    • Exact the same as above

Chapter 7: Manipulating and Receiving Mouse Input from the Camera

  • 7.1. Simple Camera Manipulations
    • WASD: move hero to push window (at 90% of the bound region)
    • Portal and Brain: are always kept within the window bound
    • L/R: pan the center of the camera to center on the corresponding objects
    • N/M: zoom with respect to the center of the window
    • J/K: zoom with respect to either Hero (H) or Portal (P)

Chapter 8: Creating Graphical and Illumination Effects for Game Objects

  • 8.1. Simple Global Ambient
    • LMB/MMB: to increase/decrease global red ambient
  • 8.2. Simple Light Shader: One Light Source
    • WASD: moves Dye
    • LMB + WASD: moves Dye with the light source
    • Left/Right arrow: increase/decrease the light intensity
    • Notice: negative light can be interesting (since we are only adding)
    • Notice: Right-Minion is not illumindated
  • 8.3. Diffuse Shader with Multiple Light Sources
    • Lights:
      • 0: Dye  + Background only
      • 1: Left Minion + Background
      • 2: Right Minion + Background
      • 3: Dye + Background
    • WASD: moves Dye
    • 0, 1, 2, 3: select light
    • Arrow: move XY of selected light position
      • Z/X: inc/dec z of the selected light
      • C/V: inc/dec inner cone
      • B/N: inc/dec outer cone
      • K/L: inc/dec intensity (try negative light!)
      • H: to toggle current selected light on/off
  • 8.4. Illumi/nation Shader: Multiple Lights with Normal Map
    • Command same as above.
  • 8.5. Materials: Diffuse and Specularity
    • Same commands as 8.3 with additional
    • 5: Selects the Left Minion
    • 6: Selects Dye
    • 7, 8, 9: Selects the Ambient, Diffuse, or Specular component of the selected character
    • E/R, T/Y, U/I: increase/decrease the R, G, or B channel of the selected component
    • O/P: increase/decrease the shinngingness
  • 8.6: Different Light Types
    • Control as the combined of 8.3 and 8.5
    • Space+Arrow: to move the X/Y of light direction
  • 8.7: Shadow Shaders
    • Background and Large Space craft (on lower right) are shadow receivers
    • Large space craft casts shadow on background
    • Left-Illuminated hero does NOT cast shadow on background
    • Command same as 8.6
