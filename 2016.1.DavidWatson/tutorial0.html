Introduction

The GTCS Game Engine consists of numerous files in a very specific file hierarchy. The details of how the files work is beyond the scope of these tutorials. In this tutorial, you will setup the file structure for your game.

The run your game, you will need an HTML shell running on a web server. This file must reference all of the files that make up the engine and the fies that are specific to your game.

The format of your HTML file can be broken down into three sections...

  • Body tag - This will have a JavaScript snippet executed onload that will create a new instance of the Scene object subclass that will be sent into the game engine for execution.
  • JavaScript include files - These will be all of the files that are part of the engine and any files you create. The order that these are listed in is important. Modifying this order could break things.
  • Canvas DOM object - This is the section of the web page where all WebGL activity will be encapsulated. It identifies the canvas' pixel width and height where the game will be rendered.

onload

The entry point for your code will be an object you design that is a subclass of the engine's "Scene" object. You will first allocate a new instance of your class and send it into the game engine as shown in the following code snippet.

<body onload="
	var mainGame = new MyGameScene();
	gEngine.Core.initializeEngineCore('GLCanvas', mainGame);
">

Note: The portions in bold are the sections that are specific to your game implementation.

Canvas item

The canvas is where the rendering of your game will occur. You need to define the pixel width and height. The engine will look for the DOM object called "GLCanvas" and handle the rest.

<canvas id="GLCanvas" width="500" height="400">
        <!-- this message will show only if WebGL failed to load-->
        Your browser does not support the HTML5 canvas.
</canvas>
